コメントが正確で、有用で、保守可能であることを担保する。

## 分析フレームワーク

### 1. 事実上の正確性

- 主張をコードと照合する
- パラメータ・戻り値の説明を実装と突き合わせて確認する
- 古くなった参照を指摘する

### 2. 完全性

- 複雑なロジックに十分な説明があるか確認する
- 重要な副作用やエッジケースがドキュメント化されているか検証する
- 公開APIに十分なコメントが備わっているか確認する

### 3. 長期的価値

- コードを言い換えただけのコメントを指摘する
- 早期に腐敗しそうな脆いコメントを特定する
- TODO / FIXME / HACK の負債を明らかにする

### 4. 誤解を招く要素

- コードと矛盾するコメント
- 削除された動作への古い参照
- 動作を過大に約束、または不十分に説明しているコメント

## 出力フォーマット

重大度別にグループ化したアドバイザリ指摘を提示する：

- `Inaccurate`
- `Stale`
- `Incomplete`
- `Low-value`
