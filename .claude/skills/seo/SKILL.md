# SEO

小手先の手法ではなく、**技術的な正しさ・パフォーマンス・コンテンツの関連性**によって検索可視性を高める。詳細は `references/` 配下を参照する。

## 起動タイミング

- クロール可能性、インデックス可能性、canonical、リダイレクトを監査するとき
- title タグ、meta description、見出し構造を改善するとき
- 構造化データを追加または検証するとき
- Core Web Vitals を改善するとき
- キーワードリサーチを行い、キーワードを URL にマッピングするとき
- 内部リンクやサイトマップ／robots の変更を計画するとき

## 主要原則

1. **コンテンツ最適化の前に、技術的な障害を解消する** — クロール／インデックスが正しくない状態で文章だけ磨いても効果は出ない
2. **1 ページにつき、明確な主要検索インテントは 1 つ** — カニバリゼーションを避ける
3. **操作的な手法より、長期的な品質シグナルを優先する** — キーワード詰め込みは現代の Google では逆効果
4. **モバイルファースト前提** — Google のインデックスはモバイルファースト
5. **推奨事項はページ単位で具体的かつ実装可能でなければならない** — 「SEO を改善」のような汎用提案は禁

## アンチパターン

| アンチパターン | 対処 |
|---|---|
| キーワードの詰め込み | まずユーザー向けに書く |
| 薄いほぼ重複ページ | 統合するか差別化する |
| 実在しないコンテンツ向けのスキーマ | スキーマを実態に一致させる |
| 実ページを確認しないコンテンツ提案 | まず実ページを読む |
| 汎用的な「SEO を改善」アウトプット | すべての推奨をページまたはアセットに紐付ける |
