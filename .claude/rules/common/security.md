# セキュリティガイドライン

## シークレット管理

- ソースコードにシークレットを絶対にハードコードしない
- 必ず環境変数またはシークレットマネージャーを使用する
- 起動時に必要なシークレットの存在を確認する
- 漏洩した可能性のあるシークレットをローテーションする

## 一般的な脆弱性への対策

代表的な脆弱性に対して以下の対策を施す：

- **SQL インジェクション**: パラメータ化クエリまたは ORM を使用し、文字列連結によるクエリ構築は行わない
- **XSS**: 出力時に HTML をサニタイズし、信頼できないデータをそのまま DOM に挿入しない
- **CSRF**: 状態変更操作には CSRF トークンまたは SameSite Cookie を用いる
- **認証・認可**: 保護リソースへのアクセスでは認証と認可を明示的に検証する
- **レート制限**: 公開エンドポイントには適切なレート制限を設ける
- **エラーメッセージ**: 機密データ・スタックトレース・内部実装の詳細をクライアントに返さない

## セキュリティ対応プロトコル

セキュリティ問題が見つかった場合：
1. 直ちに作業を中断する
2. セキュリティレビューを実施する（脆弱性スキャン、コードレビュー、必要に応じて脅威モデリング）
3. 作業を継続する前にCRITICALな問題を修正する
4. 漏洩したシークレットをローテーションする
5. コードベース全体で類似の問題がないか確認する
