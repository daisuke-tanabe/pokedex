- ブランチ戦略: GitHub Flow（`main` を統合ブランチとし、PR 経由でマージ）
- コミットメッセージ: Conventional Commits
- 言語ポリシー: コミットメッセージ・PR 本文・ドキュメントは原則日本語

## AI レビュー運用

`apps/` または `packages/` 配下の `.ts` / `.tsx` を Edit / Write したあと、
**`git commit` を打つ前に必ず** Agent ツールでレビューを並列起動する。
これにより PR 段階での大きな手戻りを防ぐ。

### 起動するエージェント

- `typescript-reviewer`: 型・非同期・設計・パフォーマンス・セキュリティ・テスタビリティ
- `react-reviewer`: React コンポーネント (`apps/web` / `apps/mobile`) を変更した場合のみ。
  Hooks ルール / 再レンダリング / RSC 境界 / アクセシビリティ等

両方を 1 つのアシスタント応答内で並列発行する。

### レビュー結果の反映

- **Critical / Major**: 該当ファイルを Edit で即修正してからコミット
- **Minor / Info**: 情報共有のみ、コミット続行

### 例外 (レビュー不要)

- 設定ファイル (`*.json` / `*.yml` / `*.toml`) のみのコミット
- ドキュメント (`*.md`) のみのコミット
- 機械的な小修正 (Dependabot による依存更新など) で TS/JS のロジック変更がない場合
- 同一の変更に対する 2 回目以降のコミット (例: review 後の修正コミット、`--amend`)

### 心得

- 同じ変更を 2 度レビューしない (履歴を踏まえて重複起動を避ける)
- レビュー結果を機械的に全部修正しない。妥当性を判断する
